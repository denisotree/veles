# 프로젝트 메모리 & 학습 루프

> 🌐 **언어:** **English** · [Русский](../../ru/explanation/project-memory-and-learning-loop.md)

Veles의 결정적인 특징은 프로젝트별로 **기억**하고 **학습**한다는 것입니다. 이 페이지는
그 메모리가 무엇인지, 그리고 학습 루프가 어떻게 유용하게 유지되는지 설명합니다.

## 메모리는 구조화된 아티팩트

프로젝트 메모리는 `<project>/.veles/`에 위치합니다 — `memory.db` (SQLite, 진실의 원천)와
사람이 읽을 수 있는 `.veles/memory/` 트리 (렌더링된 인사이트 뷰, 세션 다이제스트, 제안,
시스템 운영 저널). 이것은 **사용자 콘텐츠와 분리**되어 있으며 어떤 레이아웃(wiki, notes,
또는 bare)에서도 동일하게 작동합니다. 채팅 기록 덤프가 아닙니다 — 구조화된 레이어의 집합입니다:

- **세션 로그** — 모든 대화, 턴당 하나의 행, 전문 검색 인덱싱.
- **규칙** — 에이전트가 따라야 하는 짧은 명령어 (`format`, `do`, `don't`, `preference`),
  안정적인 시스템 프롬프트에 주입됨.
- **인사이트** — 세션에서 증류된 교훈. SQL 행이 정식 (회상, 에이징, 중복 제거가 여기서 작동);
  마크다운 뷰가 인간과 내보내기를 위해 `.veles/memory/insights/`에 렌더링됨.
- **프로젝트 트리 맵** — 에이전트가 전체 트리 대신 관련 파일 3–5개를 읽을 수 있도록
  의미론적으로 태그된 캐시된 파일 맵.
- **스킬 & 도구 레지스트리** — 랭킹과 중복 제거가 사용하는 텔레메트리 (사용/성공/오류 수)와 함께.

[프로젝트 레이아웃](../reference/project-layout.md#project-memory-velesmemorydb)의 테이블 목록을 참조하세요.

## 회상: 작은 컨텍스트, 필요 시 가져오기

`AGENTS.md`는 의도적으로 작게 유지됩니다. 무언가를 물어보면 Veles는 관련된 것만
가져옵니다: 일치하는 과거 턴 (전문 검색 + 선택적 벡터 리랭킹), 적용 가능한 규칙과 인사이트,
프로젝트 트리 맵이 가장 높게 점수를 매긴 파일들. 이것은 모든 것을 덤프하는 대신 각
모델 호출을 집중적이고 저렴하게 유지합니다.

## 학습 루프

경험은 세 가지 메커니즘을 통해 지속적인 지식이 됩니다:

### 인사이트 — 교훈 캡처
실행 후, 추출기가 기억할 가치가 있는 것을 찾습니다: 명시적인 "X를 기억해" / "절대 Y 하지마"
피드백, 그리고 도구 오류→복구 패턴 (실패 후 수정). 이것들을 인사이트와 규칙으로 증류해
같은 실수가 반복되지 않도록 합니다.

### 큐레이터 — 세션 통합
큐레이터는 오래된 세션을 지속적인 메모리로 증류합니다: 항상 SQL 인사이트와 규칙; 추가로
프로젝트 레이아웃이 wiki 엔진을 활성화할 때 `wiki/sessions/` 페이지. 유휴/턴 후 타이머에
실행되거나, `veles curate`로 즉시 실행할 수 있습니다.

### 드리밍 — 백그라운드 유지 관리
`veles dream` (그리고 유휴 시 데몬)은 인사이트를 추출하고, 스킬과 인사이트를 중복 제거하고,
승격을 제안하며, (wiki 레이아웃에서) wiki를 린트합니다 — 사용자를 차단하지 않고 메모리를
최신 상태로 유지합니다. 더 깊은 LLM 패스를 위해 `--include-consolidation`을 추가하세요.

## 컨텍스트 압축

긴 대화는 슬라이딩 윈도우 압축기에 의해 모델의 컨텍스트 한도 아래로 유지됩니다: 인메모리
기록이 토큰 임계값을 초과하면 중간 부분이 (저렴하게 라우팅된 모델로) 요약되고 `.veles/memory/sessions/`에
저장된 요약의 포인터로 대체됩니다. 전체 기록은 항상 `memory.db`에 남아 있습니다 —
인메모리 윈도우만 압축되므로 디스크에서는 무손실입니다.

## 왜 이것이 중요한가

메모리가 구조화되어 있고 루프가 지속적으로 실행되기 때문에, Veles 프로젝트는 **더 많이
사용할수록 더 유용해집니다** — 사용자의 규칙을 학습하고, 반복 오류를 피하고, 실제로 본
것에 근거하여 답변합니다.
