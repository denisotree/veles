# Справочник CLI

> 🌐 **Языки:** [English](../../en/reference/cli.md) · [简体中文](../../zh-CN/reference/cli.md) · [繁體中文](../../zh-TW/reference/cli.md) · [日本語](../../ja/reference/cli.md) · [한국어](../../ko/reference/cli.md) · [Español](../../es/reference/cli.md) · [Français](../../fr/reference/cli.md) · [Italiano](../../it/reference/cli.md) · [Português (BR)](../../pt-BR/reference/cli.md) · [Português (PT)](../../pt-PT/reference/cli.md) · **Русский** · [العربية](../../ar/reference/cli.md) · [हिन्दी](../../hi/reference/cli.md) · [বাংলা](../../bn/reference/cli.md) · [Tiếng Việt](../../vi/reference/cli.md)

Все команды, подкоманды и флаги Veles. Запустите `veles <command> --help`, чтобы
получить актуальную, всегда свежую сигнатуру — эта страница повторяет парсеры
аргументов из `src/veles/cli/_parsers/`.

```
veles [--no-wizard] <command> [subcommand] [options]
```

- `--no-wizard` — пропустить мастер первичной настройки, даже если
  `~/.veles/config.toml` отсутствует (срабатывает также при наличии TTY и при
  `VELES_NO_WIZARD=1`).
- Без аргументов `veles` запускает интерактивный [TUI](tui.md).

Большинство команд агента принимают [общие флаги цикла агента](#общие-флаги-цикла-агента)
и [имена провайдеров](#имена-провайдеров), перечисленные внизу.

## Жизненный цикл проекта

### `veles init [name]`
Создаёт новый проект Veles в текущем каталоге (служебный каталог `.veles/`
+ `AGENTS.md` + контентный каркас выбранного layout-пакета).

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `name` (позиционный) | имя каталога cwd | Имя проекта |
| `--layout <name>` | `llm-wiki` | Layout-пакет для контентного каркаса (`llm-wiki`, `notes`, `bare` или кастомный пакет из `~/.veles/layouts/`) |
| `--force` | выкл | Пересоздать `.veles/`, даже если каталог уже существует |

### `veles schema {validate,edit,fix}`
Проверяет или редактирует `AGENTS.md` (контекстный файл проекта).

- `validate` — проверить наличие обязательных секций H2.
- `edit` — открыть `AGENTS.md` в `$EDITOR` (по умолчанию `vi`), проверить при выходе.
- `fix` — интерактивно добавить недостающие секции через LLM-мастер.

### `veles self-doc [refresh|show]`
Генерирует и отображает самодокументацию проекта (`wiki/self-doc/overview.md`).
Голый `veles self-doc` показывает текущую страницу; `refresh` пересоздаёт её.

### `veles doctor`
Запускает проверки состояния по user-global состоянию и активному проекту.
Работает как с активным проектом, так и без него.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--json` | выкл | Выдать отчёт в JSON |
| `--strict` | выкл | Завершиться с ненулевым кодом при любом предупреждении (для CI-гейтинга) |

### `veles export {full,template} <path>`
Упаковывает проект в архив `.tar.gz`. См. [Резервное копирование и обмен](../how-to/backup-and-share.md).

- `full <path>` — весь проект (`.veles/` + `AGENTS.md`), без runtime-эфемерных файлов.
- `template <path>` — очищенное подмножество (схема + навыки + модули +
  не-сессионные wiki-страницы); удаляет `memory.db`, `sources/`, `sessions/`,
  trust-разрешения и редактирует текст для удаления PII.

### `veles import <path>`
Восстанавливает архив, созданный командой `veles export`.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `path` (позиционный) | — | Путь к архиву (`.tar.gz`) |
| `--into <dir>` | cwd | Целевой каталог |
| `--force` | выкл | Перезаписать существующий `.veles/` в целевом каталоге |

## Запуск агента

### `veles run "<prompt>"`
Выполняет один промпт от начала до конца с сохранением памяти и триггерами
куратора/обучения. Принимает все [общие флаги цикла агента](#общие-флаги-цикла-агента), плюс:

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--resume <session_id>` | новая сессия | Продолжить существующую сессию |
| `--manager` | выкл | Декомпозировать через мультиагентного менеджера (также `VELES_MANAGER_MODE=1`) |
| `--verify` | выкл | После запуска маршрутизированный advisor оценивает ответ; при уверенном провале перезапускает на более сильной модели (также `VELES_VERIFY_MODE=1`) |
| `--plan` | выкл | Режим планирования: чтение/поиск/черновики разрешены, изменения блокируются |
| `--no-agents-md` | выкл | Не внедрять `AGENTS.md` в системный промпт |
| `--no-index` | выкл | Не внедрять `wiki/INDEX.md` |
| `--no-compress` | выкл | Отключить компрессию контекста скользящим окном |
| `--no-curator` | выкл | Отключить триггеры куратора для этого запуска |
| `--no-insights` | выкл | Отключить извлечение инсайтов после запуска |
| `--no-proposer` | выкл | Отключить автотриггер предложения подпроектов |
| `--no-route-refresh` | выкл | Отключить обновление NL-маршрутизации из `AGENTS.md` |
| `--no-suggest-promote` | выкл | Отключить автосоветчик по продвижению |
| `--compressor-model <id>` | маршрутизированная | Переопределить модель компрессии |
| `--compress-threshold-tokens <n>` | `50000` | Размер истории, запускающий компрессию |

### `veles tui`
Открывает интерактивный REPL. См. [справочник TUI](tui.md). Принимает общие
флаги цикла агента, `--resume`, флаги внедрения/компрессии `--no-*` выше, и:

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--theme <name>` | из конфига или `everforest` | Цветовая тема (everforest, dracula, gruvbox, tokyo-night, catppuccin) |

### `veles add <source>`
Читает источник (локальный файл или `http(s)://` URL) и синтезирует его в
wiki-страницу. Принимает общие флаги цикла агента.

### `veles curate`
Запускает один проход куратора: уплотняет необработанные сессии в страницы
`wiki/sessions/`.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--limit <n>` | небольшое значение | Макс. число сессий, обрабатываемых за этот запуск |

Плюс общие флаги цикла агента.

### `veles research "<question>"`
Глубокое исследование: декомпозиция на подвопросы → параллельное изучение веба →
синтез отчёта с цитатами.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--max-subquestions <n>` | `4` | Число параллельных исследовательских направлений |

Плюс общие флаги цикла агента.

### `veles dream`
Запускает один фоновый цикл консолидации памяти (инсайты → дедуп навыков →
советы по продвижению → линтинг wiki, опционально LLM-консолидация).

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--include-consolidation` | выкл | Запустить дорогую LLM-консолидацию (нужен API-ключ) |
| `--dry-run` | выкл | Выполнить все шаги, но пропустить запись в `wiki/state` |
| `--skip-insights` / `--skip-dedup` / `--skip-promote` / `--skip-lint` | выкл | Пропустить отдельные шаги |
| `--consolidation-model <id>` | маршрутизированная (откат на `anthropic/claude-haiku-4.5`) | Переопределить модель консолидации |
| `--provider <name>` | маршрутизированный | Провайдер для подагента консолидации (опустите, чтобы использовать маршрутизированный провайдер проекта) |
| `--project-root <path>` | автоопределение | Переопределение проекта |

## Знания: навыки, инструменты, модули

### `veles skill {list,show,add,remove,promote,demote,dedup,suggest-promote}`

| Подкоманда | Назначение |
|---|---|
| `list` | Список навыков активного проекта (с телеметрией) |
| `show <name>` | Вывести `SKILL.md` навыка |
| `add <source> [--name N] [--scope project\|user] [-y]` | Установить из git-URL или локального пути |
| `remove <name> [--scope project\|user] [-y]` | Удалить установленный навык |
| `promote <name> [--keep-telemetry]` | Скопировать навык проекта в user-область (`~/.veles/skills/`) |
| `demote <name> [-y]` | Скопировать user-навык в активный проект |
| `dedup [--mode auto\|embedding\|tfidf] [--embedding-threshold f] [--tfidf-threshold f]` | Найти почти дублирующиеся навыки |
| `suggest-promote [--save] [--min-uses n] [--min-success-rate f]` | Список навыков, проходящих порог автопродвижения |

### `veles tool {list,show,promote}`

| Подкоманда | Назначение |
|---|---|
| `list` | Список инструментов, занесённых в `memory.db` этого проекта |
| `show <name>` | Вывести манифест инструмента + телеметрию |
| `promote <name> [-y]` | Переместить инструмент проекта в `~/.veles/tools/` (между проектами) |

### `veles module {list,show,add,remove}`

| Подкоманда | Назначение |
|---|---|
| `list` | Список установленных модулей |
| `show <name>` | Вывести манифест модуля |
| `add <source> [--name N] [-y]` | Установить модуль из git-URL или локального пути |
| `remove <name> [-y]` | Удалить установленный модуль |

### `veles browse {modules,skills} [query]`
Просмотр кураторских реестров.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `query` (позиционный) | `""` | Фильтр по подстроке |
| `--source <url>` | канонический | Переопределить источник реестра |
| `--json` | выкл | Выдать JSON |

### `veles project {list,add,remove,switch}`

| Подкоманда | Назначение |
|---|---|

## Маршрутизация и модели

### `veles route {show,set,reset,refresh}`
Ансамблевая маршрутизация по задачам — какой `provider:model` обрабатывает каждый
тип задачи (`default`, `curator`, `compressor`, `insights`, `skills`, `advisor`,
`vision`, `embedding`). См. [маршрутизацию по задачам](../how-to/per-task-routing.md).

| Подкоманда | Назначение |
|---|---|
| `show` | Вывести разрешённую таблицу маршрутизации для активного проекта |
| `set <task> <provider:model>` | Закрепить задачу за спецификацией |
| `reset [task]` | Сбросить одну задачу (или все) к значениям по умолчанию |
| `refresh [--force]` | Заново разобрать подсказки маршрутизации на естественном языке из `AGENTS.md` |

### `veles models <provider>`
Список моделей провайдера. Облачные провайдеры (openrouter/openai/gemini)
кэшируются на 24 часа; локальные провайдеры всегда отдают актуальные данные.

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `provider` (позиционный) | — | Один из [имён провайдеров](#имена-провайдеров) |
| `--refresh` | выкл | Обойти дисковый кэш (только облако) |
| `--json` | выкл | Выдать `{provider, source, models}` в JSON |

## Долгие задачи

### `veles goal {list,show,start,checkpoint,pause,resume,done,cancel}`
Долгосрочные цели с бюджетами и контрольными точками.

| Подкоманда | Назначение |
|---|---|
| `list [--status active\|paused\|completed\|blocked\|cancelled]` | Список целей |
| `show <id> [--json]` | Показать одну цель |
| `start "<objective>" [--scope S] [--done-when D] [--max-steps n] [--max-cost-usd f] [--max-wall-time-s n] [--forbid A]… [--approve A]…` | Создать цель |
| `checkpoint <id> "<note>" [--evidence U] [--cost-usd f] [--no-advance]` | Добавить запись о прогрессе |
| `pause <id>` / `resume <id>` | Пауза / возобновление |
| `done <id> [--evidence E]` / `cancel <id> [--reason R]` | Завершить / отменить |

### `veles job {add,list,show,pause,resume,trigger,remove,history,tick}`
Запланированные задачи агента.

| Подкоманда | Назначение |
|---|---|
| `add --name N --schedule S --prompt P [--repeat n] [--context-from JOB_ID] [--deliver-to TARGET]` | Создать задачу (schedule = cron, `<N><s\|m\|h\|d>` или ISO-метка времени) |
| `list [--json]` / `show <id>` | Просмотр задач |
| `pause <id>` / `resume <id>` / `trigger <id>` / `remove <id>` | Жизненный цикл |
| `history <id> [--limit n]` | Недавние запуски |
| `tick` | Синхронно выполнить все наступившие задачи один раз (демон не нужен; принимает флаги цикла агента) |

## Безопасность и контроль доступа

### `veles trust {list,set,revoke,clear}`
Сохранённые разрешения для чувствительных инструментов (`run_shell`, `write_file`,
`fetch_url`, …). См. [безопасность](../how-to/security-and-permissions.md).

| Подкоманда | Назначение |
|---|---|
| `list` | Показать разрешения (user- + project-область) |
| `set <tool> [--scope project\|user]` | Выдать разрешение инструменту |
| `revoke <tool> [--scope project\|user\|both]` | Снять разрешение |
| `clear [--scope project\|user\|all]` | Стереть разрешения в области |

### `veles autopilot {enable,disable,status}`
Ограниченное по времени окно, в котором запросы trust-лестницы автоматически
разрешаются.

| Подкоманда | Назначение |
|---|---|
| `enable --until <DUR>` | Открыть окно (`+30m`, `+2h`, `+1d` или ISO `2026-05-12T18:00:00Z`) |
| `disable` | Закрыть окно сейчас |
| `status` | Сообщить, активен ли автопилот |

### `veles secret {set,get,list,delete}`
Секреты на базе keychain ОС (API-ключи, токены ботов).

| Подкоманда | Назначение |
|---|---|
| `set <name> [value]` | Сохранить (опустите значение для интерактивного ввода / stdin) |
| `get <name> [--reveal] [--no-env-fallback]` | Получить (по умолчанию с откатом на env) |
| `list` | Показать, какие канонические секреты настроены |
| `delete <name>` | Удалить секрет |

## Демон и каналы

### `veles daemon [start|stop|status|list|restart|delete|session|token]`
Запуск/управление HTTP+WS-демоном. Голый `veles daemon` открывает TUI
**выбора демона** (проект → демоны → каналы). См. [запуск в виде демона](../how-to/run-as-daemon.md).

| Подкоманда | Назначение |
|---|---|
| `start [--host H] [--port P] [--foreground] [--name N]` | Запустить демон (по умолчанию отсоединяется) |
| `stop [--name N]` / `status [--name N]` | Остановить / проверить |
| `list` | Список демонов по всем проектам |
| `restart [target] [--name N]` | Остановить + перезапустить на том же хосте/порту |
| `delete <target> [-y]` | Остановить + удалить из реестра |
| `session create <name> [--host H] --port P [--model M] [--provider P] [--mode M]` | Объявить именованную сессию демона |
| `session list [--all]` / `session delete <name>` | Управление именованными сессиями |
| `token add <name>` / `token list` / `token remove <name>` | CRUD bearer-токенов |

`start` также принимает общие флаги цикла агента; для демона `--model` /
`--provider` по умолчанию берутся из конфига проекта и зафиксированы на всё время
жизни демона.

### `veles channel {list,run,list-sessions,reset-session,add,remove}`
Внешние чат-шлюзы (Telegram, …), которые общаются с демоном. См.
[подключение Telegram](../how-to/connect-telegram.md).

| Подкоманда | Назначение |
|---|---|
| `list` | Список зарегистрированных платформ каналов + число сессий |
| `run --channel telegram [--bot-token T] [--daemon-url U] [--daemon-token T]` | Запустить шлюз в переднем плане |
| `list-sessions [--channel C]` | Показать соответствия `chat_id → session_id` |
| `reset-session <chat_id> [--channel C]` | Забыть соответствие (следующее сообщение начнётся с нуля) |
| `add [--channel C] [--session S]` | Привязать канал к демону (мастер; учётные данные → keychain) |
| `remove <channel> [--session S]` | Удалить привязку канала |

## Общие флаги цикла агента

Принимаются командами `run`, `add`, `tui`, `curate`, `research`, `job tick` и
`daemon start`:

| Флаг | По умолчанию | Назначение |
|---|---|---|
| `--model <id>` | разрешается из `[provider]` model проекта → user `default_model` (жёстко зашитого значения нет) | ID модели |
| `--provider <name>` | `openrouter` | Провайдер (см. ниже) |
| `--max-tokens-total <n>` | `100000` | Совокупный бюджет токенов; `0` отключает |
| `--max-iterations <n>` | `30` | Макс. число итераций вызова инструментов за ход |
| `--stream` | выкл | Стримить ответ токен за токеном |
| `--verbose` / `-v` | выкл | Прогресс по каждому ходу в stderr |
| `--project-root <path>` | автоопределение из cwd | Работать с проектом в другом месте |
